# 标题：Outlook 日历不同步处理手册

## 适用场景

员工反馈 Outlook 日历不同步，表现为电脑端和手机端会议安排不一致，或者新建会议后长时间没有同步到其他设备。

## 常见现象

1. Outlook 桌面端能看到会议，但手机端看不到。
2. 手机端新增的日历事件没有同步到 Outlook 桌面端。
3. 会议时间已经修改，但参会人看到的仍然是旧时间。
4. Outlook 提示“正在更新此文件夹”，但长时间没有完成。

## 处理步骤

1. 先确认电脑和手机网络正常，并且 Outlook 账号已经成功登录，没有反复弹出密码框。
2. 在 Outlook 桌面端点击“发送/接收”，手动执行一次同步，再观察日历事件是否更新。
3. 检查用户登录的邮箱账号是否正确，避免把个人邮箱和企业邮箱混用。
4. 在手机端打开 Outlook 应用，进入账号设置，确认“日历同步”开关已开启。
5. 如果只有某一个共享日历不同步，检查该共享日历权限是否仍然有效。
6. 若桌面端缓存异常，可关闭 Outlook 后重新打开；如果仍不恢复，再重建 Outlook 配置文件或重新添加账号。
7. 如果会议邀请已经发出但参会人仍然看到旧时间，建议重新发送会议更新通知。

## 补充说明

- Outlook 日历不同步问题通常和网络、账号配置、客户端缓存或共享权限有关。
- 如果是多名员工同时出现 Outlook 日历不同步，优先怀疑邮件服务端或统一策略异常。

## 升级条件

- 用户已经完成标准排查，但 Outlook 日历仍然不同步。
- 同一部门多名员工同时出现日历同步异常。
- Outlook 桌面端和手机端都无法完成同步，且重新登录后问题仍然存在。
