标题：Git 仓库权限异常排查

适用场景
员工执行 git clone、git pull、git push 时提示权限不足或仓库不存在。

处理步骤
1. 确认使用的是公司要求的 Git 账号。
2. 检查本机 SSH 公钥是否已经上传到代码平台。
3. 执行 SSH 连通性测试，确认密钥认证是否通过。
4. 核对仓库地址是否为正确的 HTTPS 或 SSH 地址。
5. 如果是新加入项目成员，确认项目管理员是否已授予仓库权限。
6. 若 push 被拒绝，检查当前分支是否受保护。
7. 如果确认账号和仓库地址都正确但仍无权限，请联系管理员或创建工单。

补充说明
- 常见原因是 SSH Key 未绑定或仓库角色权限未开通。
